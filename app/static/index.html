What is Izanagi?

Izanagi is a self-hosted API service that lets coding agents — Claude Code, Aider, Cursor, or any tool that can make HTTP requests — deploy Unraid Community Applications XML templates directly to your server and trigger container restarts, without you touching the Unraid UI.

Every deploy is recorded in history with a timestamp, the agent that triggered it, and the outcome. Access is controlled by per-agent bearer tokens so you can see exactly who deployed what, and revoke access individually.

Sections

Deploy

Paste or upload an Unraid XML template and give it a name. If you also provide a container name, Izanagi will attempt a Docker restart immediately after the file is written. If the restart is not possible (e.g. the container needs to be fully recreated), the UI will tell you and link directly to the Unraid Docker page.

History

A log of every deploy: timestamp, template name, which agent triggered it, whether the template was newly created or updated, and whether the deploy succeeded or failed.

Tokens

Create and revoke bearer tokens. Each token is tied to an agent name for attribution. The raw token value is shown exactly once at creation — copy it immediately. Tokens are stored as SHA-256 hashes; the original value is never persisted.

Settings

Paste your bearer token here so the browser can make authenticated API calls. The token is stored in localStorage and never leaves your browser. Server build info is also displayed here.
